## Component breakdown
### Agent
Basic agent that interacts with the system
Has access to tools, is permission controlled etc. 

Configure by editing:
-  agent/agent_name.md
-  agent/heartbeat/agent_name.md

### Composer
In charge of animating the agents, and "orchestrating" the group. 
This will read from `sublimate-compose.yml` and permission control agents through
- Read/write controls
- Tool access

It'll also manage the heartbeats, and some basic provider syntax.
However, crucially - **the sublimate-compose.yml will not contain api keys** (for security)

So the fastapi server will have to manage this. Or this should be solved by using env keys.
I don't exactly know how this will be sorted - but the `fetch_api_keys_for_provider` function should be set for this to work. (unfinished)

The configuration file can be generated by the server.

### Server/backend 
FastAPI server that manages:
- Basic API endpoints
- Multiple project management
  - editing `sublimate-compose.yml`
  - applying different agent presets
  - editing and managing agents/ folder
- Provider management
  - Manage AI providers 
- User management
  - Password hashing
  - Project creation and editing
  - Project sharing?
- Possibly Jinja2Templates for the webui as prototype webui. Planning on adding a react client for long term.

There should also be a database that this connects to. We'll connect it with sqlalchemy so that we can swap it out for a postgres server.
